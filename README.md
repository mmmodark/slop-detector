# 전지적독자시점 (Reader Review)

> **당신 글, 솔직하게 읽어드릴게요**

열심히 쓰고 있는데 반응이 없는 사람들을 위한 AI 글 진단 서비스.  
주변 지인은 "좋아 보여"밖에 못 해줘요. 이 서비스는 독자 입장에서 솔직하게 읽어드립니다.

---

## 어떤 사람을 위한 서비스인가요?

- 블로그·링크드인·인스타·스레드에 6개월 이상 꾸준히 쓰고 있는데
- 팔로워가 늘지 않거나, 저장·댓글 반응이 없거나
- "왜 안 닿지?"를 혼자 고민하고 있는 사람

이 서비스가 풀어주는 건 **글 자체의 문제**예요.  
알고리즘·해시태그·팔로워 수 같은 유통 문제는 다루지 않아요.

---

## 주요 기능

### 채널별 진단
블로그·링크드인·인스타그램·스레드, 채널마다 다른 기준 5가지로 읽어요.  
"이 채널 독자라면 어디서 이탈할까?"를 잡아냅니다.

| 채널 | 진단 기준 |
|------|-----------|
| 블로그/브런치 | 훅과 도입부, 작가 vs 독자, 구조와 흐름, 그래서 나는 뭘?, 구체성 |
| 링크드인 | 첫 줄 훅, 전문성 vs 자기자랑, 그래서 나는 뭘?, 댓글·저장 욕구, 구체성 |
| 인스타그램 | 첫 줄 훅, 간결함, 그래서 나는 뭘?, 저장·공유 욕구, 진정성 |
| 스레드 | 한 방 임팩트, 날것의 진정성, 대화 욕구, 간결함, 포인트 명확성 |

### 문단별 고쳐쓰기
문제 있는 문단을 직접 고쳐서 보여줘요.  
원문과 제안을 나란히 보고 복사할 수 있어요.

### 내 패턴 분석
글을 쌓아가다 보면, 내가 매번 어디서 독자를 잃는지 패턴이 보여요.  
2편 이상 진단하면 분석 가능. 3편 이상 선택할수록 정확도가 올라가요.  
분석 리포트는 PDF로 저장할 수 있어요.

## 기술 스택

- **프론트엔드:** HTML/CSS/JS 단일 파일 (`index.html`)
- **백엔드:** Vercel 서버리스 함수 (`api/diagnose.js`)
- **AI:** Claude API (`claude-sonnet-4-6`)
- **배포:** GitHub → Vercel 자동배포
- **인증:** 비밀코드 방식 (서버에서 ACCESS_CODE 검증)
- **PDF 저장:** jsPDF (cdnjs CDN)

---

slop-detector/

Vercel 환경변수에만 존재하기 때문에 GitHub에 통째로 올려도 안전해요.

## 배포 방법

### 1단계 — GitHub에 올리기

1. `github.com`에서 새 저장소를 만든다
2. 이 폴더 파일을 그 저장소에 올린다 (드래그 업로드 or git 명령어)

### 2단계 — Vercel에 연결하기

1. `vercel.com`에 GitHub 계정으로 로그인한다 (무료)
2. **Add New… → Project** 클릭
3. 저장소 선택 → Import
4. 설정은 그대로 두고, 아래 3단계 먼저 진행 후 Deploy

### 3단계 — 환경변수 넣기 ★ 가장 중요 ★

Vercel 프로젝트 → **Settings → Environment Variables** 에서 두 개 추가:

| Key | Value |
|-----|-------|
| `ANTHROPIC_API_KEY` | Anthropic API 키 (`sk-ant-...`) |
| `ACCESS_CODE` | 원하는 비밀코드 (예: `abc2025`) |

- 철자 그대로 입력 (대문자, 밑줄 포함)
- 저장 후 반드시 **Redeploy** 클릭

### 4단계 — 확인

배포 후 Vercel이 준 URL로 접속 → 비밀코드 입력 → 진단기 열림

## 자주 나는 문제

| 증상 | 원인 | 해결 |
|------|------|------|
| 입장이 안 됨 | `ACCESS_CODE` 불일치 또는 Redeploy 미완료 | 환경변수 확인 후 Redeploy |
| 진단 눌렀는데 통신 실패 | API 키 오류 또는 크레딧 소진 | `ANTHROPIC_API_KEY` 및 잔액 확인 |
| 화면은 뜨는데 진단만 안 됨 | 거의 항상 환경변수 문제 | 위 두 개 재확인 |
| PDF 저장 안 됨 | jsPDF CDN 로딩 전 클릭 | 잠시 후 다시 클릭 |

## 비밀코드 바꾸는 법

Vercel → Settings → Environment Variables → `ACCESS_CODE` 값 수정 → Redeploy
